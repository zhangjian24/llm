# RAG 文档问答系统 - 软件需求规格说明书 (SRS)

## 1. 引言

### 目的
本文档定义 RAG 文档问答系统的完整软件需求，包括功能需求、非功能需求、接口需求和数据规范，为系统设计、开发测试和验收提供基准。

### 范围
本系统是一个基于检索增强生成（RAG）技术的智能文档问答平台，支持多格式文档上传、语义检索和自然语言交互。

**系统边界**:
- ✅ 包含：文档处理、向量化、存储、检索、对话生成
- ❌ 不包含：用户权限管理、移动端应用、自定义模型训练

### 定义与缩写
| 术语 | 全称 | 说明 |
|------|------|------|
| RAG | Retrieval-Augmented Generation | 检索增强生成技术 |
| LLM | Large Language Model | 大型语言模型 |
| Embedding | 文本嵌入 | 将文本转换为向量表示 |
| Rerank | Re-ranking | 重排序，优化检索结果相关性 |
| Pinecone | - | 托管型向量数据库服务 |
| qwen-max | - | 阿里云通义千问最大模型 |
| API | Application Programming Interface | 应用程序接口 |

## 2. 总体描述

### 用户特征
**目标用户画像**:
- **企业知识管理员**: 管理企业内部文档库，快速查找政策、流程等信息
- **研究人员/学生**: 阅读大量论文、报告，需要快速定位关键信息
- **法律/财务专业人士**: 处理合同、法规文档，需要精确引用来源
- **技术支持工程师**: 查阅产品手册、故障排查指南，快速响应客户问题

**技术能力假设**:
- 具备基本的计算机操作能力
- 熟悉 Web 浏览器使用
- 无需编程或 AI 专业知识

### 运行环境
**后端环境**:
- Python 3.9+
- FastAPI 0.100+
- LangChain v1.x+
- 内存：≥4GB RAM
- 存储：≥10GB（取决于文档量）

**前端环境**:
- React 19+
- TypeScript 5+
- Node.js 18+
- 现代浏览器（Chrome 100+, Firefox 100+, Safari 16+）

**网络要求**:
- 稳定的互联网连接（用于调用阿里云百炼 API 和 Pinecone）
- 推荐带宽：≥10 Mbps

### 假设与依赖
**外部依赖**:
- 阿里云百炼 API（qwen-max、text-embedding-v4、rerank-v3）
- Pinecone Serverless 向量数据库
- OpenAI 兼容 SDK（langchain_openai）

**关键假设**:
- API 服务可用性 ≥99%
- 文档内容不包含极端敏感信息（如军事机密）
- 单次对话上下文长度不超过模型限制（32K tokens）

## 3. 功能需求 (Functional Requirements)

### FR-001: 文档上传
**名称**: 多格式文档上传  
**优先级**: Must have  
**描述**: 用户上传本地文档至系统进行处理  
**前置条件**: 用户已打开系统 Web 界面  
**后置条件**: 文档被解析、分块并向量化存储  

**输入**:
- 文件选择器选择一个或多个文件
- 支持格式：PDF、DOCX、Markdown、TXT
- 单文件大小限制：≤50MB

**处理逻辑**:
1. 验证文件格式和大小
2. 读取文件内容（使用相应解析器）
3. 提取纯文本（去除格式、图片等非文本元素）
4. 触发向量化流程（见 FR-002）

**输出**:
- 成功：显示"上传成功"，列出文档元信息（名称、页数、字数）
- 失败：显示错误原因（格式不支持、文件损坏等）

**验收标准** (Given-When-Then):
- Given 用户选择了一个有效的 PDF 文件
- When 点击"上传"按钮
- Then 系统应在 30 秒内完成处理并显示成功提示

---

### FR-002: 文档分块与向量化
**名称**: 智能文档分块与向量化  
**优先级**: Must have  
**描述**: 将文档切分为合适大小的块并转换为向量  
**前置条件**: 文档已成功上传  
**后置条件**: 向量数据存储至 Pinecone  

**输入**:
- 文档纯文本内容
- 分块配置参数（块大小、重叠率）

**处理逻辑**:
1. 按语义边界分块（段落、章节）
2. 每块大小控制在 500-1000 tokens
3. 相邻块之间保留 10-20% 重叠
4. 调用 text-embedding-v4 生成向量（维度 1536）
5. 存储至 Pinecone（包含元数据：文档 ID、块索引、原文）

**输出**:
- 向量数据存储成功确认
- 分块统计信息（总块数、平均长度）

**验收标准**:
- Given 一篇 10 页的文档
- When 完成向量化处理
- Then 应生成 20-50 个向量块，且每个块可独立检索

---

### FR-003: 语义检索
**名称**: 基于向量的语义搜索  
**优先级**: Must have  
**描述**: 根据用户查询检索最相关的文档块  
**前置条件**: 系统中已有向量化的文档  
**后置条件**: 返回相关文档块列表  

**输入**:
- 用户查询文本（自然语言）
- 检索数量配置（默认 top_k=5）

**处理逻辑**:
1. 调用 text-embedding-v4 将查询转换为向量
2. 在 Pinecone 中进行相似度搜索（余弦相似度）
3. 获取 top_k 个最相似的文档块
4. 调用 rerank-v3 对结果重排序
5. 过滤低相关性结果（阈值≥0.7）

**输出**:
- 排序后的文档块列表（包含原文、相似度分数、来源）

**验收标准**:
- Given 用户提问"如何申请年假？"
- When 系统执行检索
- Then 前 3 个结果应与"年假申请流程"高度相关（人工评估≥0.8）

---

### FR-004: 智能问答生成
**名称**: 基于 RAG 的回答生成  
**优先级**: Must have  
**描述**: 结合检索结果和用户问题生成自然语言回答  
**前置条件**: 已完成语义检索（FR-003）  
**后置条件**: 用户获得带引用来源的回答  

**输入**:
- 用户原始问题
- 检索到的文档块（top 3-5 个）
- 对话历史（可选）

**处理逻辑**:
1. 构建 Prompt：系统指令 + 检索内容 + 用户问题
2. 调用 qwen-max 生成回答
3. 强制模型标注引用来源（如"[文档 A, 第 3 段]"）
4. 流式输出（逐字显示，提升体验）

**输出**:
- 自然语言回答（包含引用标注）
- 支持 Markdown 格式（列表、代码块等）

**验收标准**:
- Given 用户提问"公司的报销政策是什么？"
- When 系统生成回答
- Then 回答必须包含具体政策条款，并标注出自哪个文档的哪一部分

---

### FR-005: 对话历史管理
**名称**: 多轮对话上下文维护  
**优先级**: Must have  
**描述**: 保存和加载对话历史，支持上下文理解  
**前置条件**: 用户已开始一次对话  
**后置条件**: 对话记录被保存并可追溯  

**输入**:
- 每轮对话的用户问题和系统回答
- 会话 ID（自动生成）

**处理逻辑**:
1. 每轮对话保存至临时存储（Session Storage）
2. 维护最近 10 轮对话（超出则丢弃最早记录）
3. 支持用户查看历史对话列表
4. 支持清空当前会话

**输出**:
- 对话历史列表（时间戳、问题摘要）
- 点击历史记录可恢复上下文

**验收标准**:
- Given 用户进行了 5 轮对话
- When 刷新页面
- Then 之前的对话记录应保持（除非主动清空）

---

### FR-006: 流式响应展示
**名称**: 打字机效果的流式输出  
**优先级**: Must have  
**描述**: 实时显示 AI 生成的内容，降低等待焦虑  
**前置条件**: qwen-max 正在生成回答  
**后置条件**: 完整回答显示在界面上  

**输入**:
- 服务端推送的文本片段（Server-Sent Events）

**处理逻辑**:
1. 建立 SSE 连接
2. 接收 token 流并逐字追加到界面
3. 显示"思考中..."状态指示器
4. 生成完成后显示"复制"、"点赞/点踩"按钮

**输出**:
- 逐字显示的动态文本
- 生成状态指示器（加载中/完成）

**验收标准**:
- Given 一个长度为 500 字的回答
- When 开始生成
- Then 用户应在 1 秒内看到第一个字，之后持续流畅显示直至完成

---

### FR-007: 文档列表与管理
**名称**: 已上传文档浏览  
**优先级**: Should have  
**描述**: 查看所有已上传的文档及其状态  
**前置条件**: 系统中已有至少一个文档  
**后置条件**: 用户可删除或重新处理文档  

**输入**:
- 文档元数据列表（从数据库查询）

**处理逻辑**:
1. 查询所有文档记录
2. 显示关键信息（名称、上传时间、状态、块数）
3. 提供操作按钮（删除、重新处理、下载原文）

**输出**:
- 文档列表表格（支持分页）
- 筛选功能（按状态、时间排序）

**验收标准**:
- Given 系统中有 10 个文档
- When 用户打开文档列表页
- Then 应显示所有文档，并支持按上传时间倒序排列

---

### FR-008: 引用来源追溯
**名称**: 回答来源高亮与跳转  
**优先级**: Should have  
**描述**: 点击引用标注可跳转到原文位置  
**前置条件**: AI 回答中包含引用标注  
**后置条件**: 用户可查看原始文档片段  

**输入**:
- 引用元数据（文档 ID、块索引）

**处理逻辑**:
1. 解析回答中的引用标记（如"[1]"）
2. 渲染为可点击链接
3. 点击后弹窗显示原文片段
4. 高亮匹配关键词

**输出**:
- 原文片段展示（只读）
- 来源文档元信息

**验收标准**:
- Given 回答中包含引用"[1]"
- When 用户点击该引用
- Then 应在侧边栏或弹窗中显示对应的原始文档内容

---

### FR-009: 搜索建议与自动补全
**名称**: 智能搜索提示  
**优先级**: Nice to have  
**描述**: 用户输入时提供相关问题建议  
**前置条件**: 用户开始在输入框中输入  
**后置条件**: 显示相关问题列表  

**输入**:
- 用户已输入的文本片段（≥3 个字符）

**处理逻辑**:
1. 监听输入框变化（防抖 300ms）
2. 调用 LLM 生成 3-5 个相关问题
3. 显示为下拉建议列表
4. 点击建议自动填充并搜索

**输出**:
- 相关问题列表（最多 5 条）

**验收标准**:
- Given 用户输入"如何请假"
- When 暂停输入超过 300ms
- Then 应显示"病假申请流程"、"年假剩余天数查询"等相关问题

### FR-010: 文档批量导入
**名称**: 文件夹批量上传  
**优先级**: Nice to have  
**描述**: 一次性上传整个文件夹的文档  
**前置条件**: 用户本地有包含多个文档的文件夹  
**后置条件**: 所有文档进入处理队列  

**输入**:
- 文件夹选择（递归遍历子文件夹）
- 文件类型过滤器（仅选择支持的格式）

**处理逻辑**:
1. 递归扫描文件夹获取文件列表
2. 过滤不支持的格式
3. 并行上传（最多同时 5 个文件）
4. 显示进度条和失败重试机制

**输出**:
- 批量上传进度面板
- 成功/失败统计

**验收标准**:
- Given 一个包含 20 个 PDF 的文件夹
- When 选择该文件夹并确认上传
- Then 系统应自动处理所有文件，成功率≥95%

## 4. 非功能需求 (Non-Functional Requirements)

### NFR-001: 性能需求
**响应时间**:
- 文档上传处理：≤30 秒/文件（50 页以内）
- 语义检索：≤2 秒（从查询到返回结果）
- 回答生成：首字延迟≤1 秒，生成速度≥50 tokens/秒
- 页面加载：≤3 秒（首次访问）

**吞吐量**:
- 支持并发用户数：≥50
- 每秒查询数（QPS）：≥10

**资源利用率**:
- CPU 平均使用率：≤70%
- 内存使用：≤4GB（空闲时≤1GB）

---

### NFR-002: 安全性需求
**认证与授权**:
- MVP 版本：无需登录（公开访问）
- 未来扩展：支持 OAuth2/JWT Token 认证

**数据加密**:
- 传输层：HTTPS/TLS 1.3
- 静态数据：Pinecone 向量数据加密存储

**输入验证**:
- 文件上传：严格限制格式和大小，防止恶意文件
- 查询输入：过滤 SQL 注入、XSS 攻击载荷
- 速率限制：单 IP 每分钟≤60 次查询

**审计日志**:
- 记录所有文档上传/删除操作
- 记录异常访问行为（频繁失败、大流量请求）

---

### NFR-003: 可靠性需求
**可用性**:
- 系统正常运行时间：≥99%（每月宕机≤7.2 小时）
- API 调用失败自动重试：最多 3 次，间隔指数退避

**容错性**:
- 单个文档处理失败不影响其他文档
- Pinecone 连接失败时降级为关键词检索

**可恢复性**:
- 文档元数据每日自动备份
- 支持从备份恢复向量数据（RPO≤24 小时）

---

### NFR-004: 可维护性需求
**日志记录**:
- 使用结构化日志（JSON 格式）
- 日志级别：DEBUG、INFO、WARNING、ERROR
- 关键操作日志保留≥90 天

**监控告警**:
- API 响应时间超过阈值（5 秒）触发告警
- 错误率超过 5% 触发告警
- 磁盘使用率超过 80% 触发告警

**代码质量**:
- 单元测试覆盖率：≥80%
- 核心模块（文档解析、向量化）覆盖率：≥95%
- 代码审查：所有 PR 需至少 1 人 Review

---

### NFR-005: 可扩展性需求
**水平扩展**:
- 支持部署多个后端实例（通过负载均衡分发请求）
- Pinecone 自动扩展（Serverless 架构）

**模块化设计**:
- 文档解析器可插拔（新增格式只需添加新解析器）
- 模型提供商可切换（支持切换到 OpenAI、Anthropic 等）

**配置化**:
- 所有魔法数字（块大小、阈值等）通过环境变量配置
- 支持多环境配置（dev/staging/prod）

---

### NFR-006: 用户体验需求
**界面友好性**:
- 遵循 WCAG 2.1 AA 无障碍标准
- 支持键盘导航（Tab 键切换焦点）
- 关键操作有明确反馈（成功/失败提示）

**响应式设计**:
- 适配桌面端（1920x1080、1366x768）

### 软件接口 (API)

#### 内部 API 端点

**POST /api/documents/upload**
- **描述**: 上传单个文档
- **请求头**: `Content-Type: multipart/form-data`
- **请求体**: 
  ```json
  {
    "file": File (binary),
    "metadata": {
      "title": string,
      "tags": string[]
    }
  }
  ```
- **响应** (200 OK):
  ```json
  {
    "id": "doc_abc123",
    "filename": "report.pdf",
    "status": "processing",
    "chunks_count": null
  }
  ```

**GET /api/documents**
- **描述**: 获取文档列表
- **查询参数**: `page=1&limit=20&sort=-created_at`
- **响应** (200 OK):
  ```json
  {
    "total": 45,
    "items": [
      {
        "id": "doc_abc123",
        "filename": "report.pdf",
        "status": "ready",
        "chunks_count": 32,
        "created_at": "2026-03-04T10:00:00Z"
      }
    ]
  }
  ```

**DELETE /api/documents/{id}**
- **描述**: 删除文档
- **路径参数**: `id` (文档 ID)
- **响应** (204 No Content)

**POST /api/chat**
- **描述**: 发起对话（流式）
- **请求头**: `Content-Type: application/json`
- **请求体**:
  ```json
  {
    "query": "如何申请年假？",
    "top_k": 5,
    "stream": true,
    "conversation_id": "conv_xyz789"
  }
  ```
- **响应** (SSE Stream):
  ```
  data: {"token": "根"}
  data: {"token": "据"}
  data: {"token": "公"}
  ...
  data: {"done": true}
  ```

**GET /api/conversations**
- **描述**: 获取对话历史列表
- **响应** (200 OK):
  ```json
  {
    "items": [
      {
        "id": "conv_xyz789",
        "last_query": "如何申请年假？",
        "turns": 5,
        "updated_at": "2026-03-04T12:30:00Z"
      }
    ]
  }
  ```

#### 外部 API 依赖

**阿里云百炼 API**:
- Base URL: `https://dashscope.aliyuncs.com/api/v1`
- 端点:
  - `/services/aigc/text-generation/generation` (qwen-max)

**Pinecone API**:

### 验收测试场景示例

**场景 1: 上传员工手册并问答**
```gherkin
Given 用户有一份"员工手册.pdf"(共 50 页)
When 用户上传该文档并等待处理完成
Then 系统应生成约 80-120 个向量块

When 用户提问"迟到扣款标准是什么？"
Then 系统应在 2 秒内返回答案
And 答案必须包含具体扣款金额或比例
And 答案必须标注出自"员工手册.pdf"第 X 章第 Y 节
```

**场景 2: 多轮对话上下文理解**
```gherkin
Given 用户已进行一轮对话:"如何申请年假？"
And 系统回答了申请流程

When 用户追问"需要提前几天申请？"
Then 系统应理解"提前几天"指的是"年假申请"
And 返回准确的提前通知期限
```

**场景 3: 低相关性查询处理**
```gherkin
Given 系统中只有技术文档（无 HR 相关内容）

When 用户提问"公司食堂菜单是什么？"
Then 系统应回复"抱歉，未在文档中找到相关信息"
And 不应编造答案（避免幻觉）
```
