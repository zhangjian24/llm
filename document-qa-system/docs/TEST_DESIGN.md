# RAG 文档问答系统 - 测试设计文档

## 1. 测试概述

### 1.1 测试目标
对 RAG 文档问答系统的前端功能进行全面的端到端测试，验证系统的可用性、稳定性和用户体验。

### 1.2 测试范围
| 类别 | 覆盖模块 |
|------|----------|
| 前端 | http://localhost:5173 |
| 后端API | http://localhost:8000 |
| 测试方式 | MCP Playwright（交互式手动测试） |

### 1.3 测试环境
- **测试框架**: MCP Playwright
- **浏览器**: Chrome (Headless)
- **测试类型**: E2E 端到端测试

## 3. 测试执行方式

### 3.1 MCP Playwright 使用指南

测试使用 MCP Playwright 工具进行，**不使用脚本测试**，而是通过交互式方式手动执行每个测试用例。

#### 启动测试
```bash
# 使用 playwright_browser_navigate 打开测试页面
playwright_browser_navigate(url="http://localhost:5173")
```

#### 测试操作类型
| 操作 | MCP 工具 |
|------|----------|
| 页面导航 | `playwright_browser_navigate` |
| 点击元素 | `playwright_browser_click` |
| 输入文本 | `playwright_browser_type` |
| 填写表单 | `playwright_browser_fill_form` |
| 截图 | `playwright_browser_take_screenshot` |
| 获取页面快照 | `playwright_browser_snapshot` |
| 等待 | `playwright_browser_wait_for` |
| 浏览器调整大小 | `playwright_browser_resize` |

#### 截图命名规范
```
screenshots/
├── T01_page_load_documents.png
├── T02_upload_pdf_success.png
├── T03_upload_docx_success.png
├── ...
└── T30_concurrent_operations.png
```
