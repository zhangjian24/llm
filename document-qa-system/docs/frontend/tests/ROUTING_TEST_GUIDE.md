# 路由测试指南

## 🎯 测试目标

验证 React Router 路由功能是否正常工作。

## 🚀 快速测试步骤

### 1. 启动开发服务器

```bash
cd frontend
npm run dev
```

**预期输出**:
```
VITE v7.3.1  ready in 3039 ms
➜  Local:   http://localhost:5173/
```

### 2. 访问应用

打开浏览器，访问：http://localhost:5173/

### 3. 测试导航功能

#### 测试用例 1: 根路径重定向
- **操作**: 访问 http://localhost:5173/
- **预期**: 显示聊天页面（ChatPage）
- **验证点**: 
  - 主内容区显示"智能问答"标题
  - 侧边栏"💬 对话"链接高亮（蓝色背景）

#### 测试用例 2: 直接访问聊天页面
- **操作**: 访问 http://localhost:5173/chat
- **预期**: 显示聊天页面
- **验证点**: 
  - URL 为 /chat
  - "💬 对话"链接高亮

#### 测试用例 3: 直接访问文档页面
- **操作**: 访问 http://localhost:5173/documents
- **预期**: 显示文档管理页面
- **验证点**: 
  - URL 为 /documents
  - "📄 文档"链接高亮
  - 主内容区显示文档列表表格

#### 测试用例 4: 点击导航切换
- **操作**: 
  1. 在聊天页面，点击"📄 文档"链接
  2. 在文档页面，点击"💬 对话"链接
- **预期**: 
  - 页面正确切换
  - 高亮状态跟随当前路径
  - 浏览器历史记录正确更新（可以后退）

#### 测试用例 5: 浏览器前进后退
- **操作**:
  1. 从 /chat 导航到 /documents
  2. 点击浏览器后退按钮
  3. 点击浏览器前进按钮
- **预期**:
  - 页面正确响应前进/后退
  - 高亮状态正确更新

## 🔍 视觉验证清单

## 🛠️ 开发者工具验证

### Chrome DevTools 检查

1. **打开开发者工具** (F12)

2. **检查当前路由**
   ```javascript
   // 在 Console 中执行
   window.location.pathname
   ```
   - 访问 /chat 时应返回 "/chat"
   - 访问 /documents 时应返回 "/documents"

3. **检查 React 组件树**
   - 打开 React Developer Tools
   - 验证 Layout、Navigation、ChatPage/DocumentsPage 组件正确渲染

4. **检查网络请求**
   - 打开 Network 标签
   - 访问 /documents 时应触发文档列表 API 调用
   - 验证 WebSocket 连接建立

## 📊 常见问题排查

### 问题 1: 页面空白
**可能原因**: 
- React Router 未正确安装
- 路由配置错误

**解决方案**:
```bash
# 检查依赖
npm list react-router-dom

# 重新安装
npm install react-router-dom
```

### 问题 2: 导航不工作
**可能原因**:
- Link 组件导入错误
- 路由路径配置错误

**解决方案**:
- 检查 main.tsx 中的 Routes 配置
- 验证 Navigation 组件中的 to 属性值

### 问题 3: 高亮状态不正确
**可能原因**:
- useLocation 未正确使用
- activeTab 逻辑错误

**解决方案**:
```typescript
// 检查 Navigation 组件
const location = useLocation();
const activeTab = location.pathname === '/documents' ? 'documents' : 'chat';
```

### 问题 4: 数据未加载
**可能原因**:
- App 组件未正确初始化
- useEffect 依赖错误

**解决方案**:
- 检查 App.tsx 中的 loadDocuments 调用
- 验证 WebSocket 连接状态

## ✅ 验收标准

所有测试用例通过，且满足以下条件:

1. ✅ 所有路由路径都能正确访问

3. ✅ 浏览器前进后退功能正常
4. ✅ 页面数据正确加载
5. ✅ 无控制台错误
6. ✅ 响应式布局正常
