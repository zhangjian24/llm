## 一、项目现状总结

### 1.1 技术栈

| 层级 | 技术 | 当前配置 |
|------|------|----------|
| **LLM** | qwen-max | 阿里云百炼 |
| **Embedding** | text-embedding-v4 | 1024维 |
| **向量库** | pgvector | HNSW索引 (M=16, ef=64) |
| **Rerank** | rerank-v3 | 阿里云百炼 |
| **分块** | 语义分块器 | size=800, overlap=150 |
| **数据库** | PostgreSQL | 元数据+向量 |

### 1.2 已有能力

- ✅ 语义分块（段落+句子）
- ✅ 向量检索（余弦相似度）
- ✅ 重排序
- ✅ 流式生成
- ✅ 对话历史
- ✅ HNSW索引
- ✅ 配置化管理

### 1.3 待改进

- ❌ 系统化评估体系
- ❌ 生产监控
- ❌ 成本追踪
- ❌ 混合搜索
- ❌ 缓存机制
- ❌ 模型路由

## 三、阶段划分

### Phase 1: 评估体系建设（第1-2周）

| 任务 | 产出 |
|------|------|
| 安装评估依赖 | ragas, deepeval |
| 创建评估模块 | backend/app/evaluation/ |
| 构建Golden数据集 | 50条人工标注 |
| 执行基线评估 | 当前指标报告 |

### Phase 2: 基础优化（第3-4周）

| 任务 | 预期效果 |
|------|---------|
| Prompt优化 | 20-30%成本降低 |
| Redis缓存 | 30-50%缓存命中 |
| 语义分块增强 | 检索质量提升 |

### Phase 3: 高级优化（第5-8周）

| 任务 | 预期效果 |
|------|---------|
| pgvector HNSW调优 | 召回率提升 |
| 混合搜索 | 向量+BM25融合 |
| 模型路由 | 30-50%成本降低 |

### Phase 4: 生产就绪（第9-12周）

| 任务 | 产出 |
|------|------|
| 监控体系 | Prometheus + Grafana |
| 成本追踪 | 每日成本报表 |
| CI/CD集成 | 质量门禁 |

## 四、效果预估

### 4.1 质量提升

| 指标 | Phase 1基线 | 最终目标 |
|------|-------------|----------|
| Faithfulness | ~0.72 | ≥0.80 |
| Answer Relevance | ~0.68 | ≥0.75 |
| Context Precision | ~0.65 | ≥0.70 |
| Context Recall | ~0.70 | ≥0.75 |

### 4.2 成本优化

| 优化项 | 节省比例 | 实施阶段 |
|--------|---------|----------|
| Prompt优化 | 20-30% | Phase 2 |
| 缓存机制 | 30-50% | Phase 2 |
| 模型路由 | 30-50% | Phase 3 |
| **总计** | **50-70%** | |

### 4.3 性能提升

| 指标 | 优化前 | Phase 3目标 |
|------|--------|------------|
| P95检索延迟 | ~320ms | <100ms |
| P95生成延迟 | ~2.8s | <2.0s |
| 缓存命中 | 0% | >30% |
