# Web E2E Testing Skill

## 测试范围矩阵

### 模块选择清单
- [ ] **布局与渲染测试** - 页面元素、样式、响应式
- [ ] **表单验证测试** - 必填、格式、长度、一致性
- [ ] **交互测试** - 点击、输入、选择、弹窗、Toast
- [ ] **导航测试** - 路由跳转、前进后退、刷新、路由守卫
- [ ] **E2E 流程测试** - 完整用户旅程
- [ ] **边界测试** - 输入边界、操作边界、并发操作
- [ ] **安全测试** - XSS、SQL注入、认证安全
- [ ] **API 集成测试** - 请求验证、响应验证、错误处理

## 详细检查清单

### 一、布局与渲染测试
每个页面验证：页面标题、核心文本、表单元素、导航元素、媒体元素、布局结构
样式验证：背景色/渐变、卡片样式、按钮样式、输入框样式、字体与对比度

### 二、表单验证测试
- 必填字段：留空提交 → 显示错误提示
- 格式校验：非法格式 → 显示错误
- 长度限制：超出范围 → 显示错误
- 一致性校验：输入不一致 → 显示错误
- 唯一性校验：重复值 → 后端错误展示

### 三、交互测试
- 用户操作：点击、输入、选择、悬停、键盘
- 状态反馈：Loading、Toast、Modal、错误高亮、成功状态

### 四、导航测试
- 路由跳转：页面内导航、跨页面导航、前进后退、刷新、直接访问URL、404
- 路由守卫：未登录→登录页、已登录→首页、无权限→403、Token过期→登录

### 五、E2E 流程测试
典型用户旅程：访问→注册→登录→使用→退出
验证点：操作反馈、页面跳转、状态传递、错误恢复、幂等性

### 六、边界测试
- 输入边界：最小/最大长度、空字符串、纯空格、特殊字符、Emoji、超长文本
- 操作边界：快速重复点击、网络断开、慢网络、并发操作、长时间停留

### 七、安全测试
- 输入安全：XSS、SQL注入、HTML注入、路径遍历
- 认证安全：Token存储、Token传递、敏感信息、控制台泄露、密码可见

### 八、API 集成测试
- 请求验证：URL、方法、请求头、请求体、响应状态码、响应体
- 错误处理：400/401/403/404/500、网络超时

## 截图规范

### 截图时机
1. 页面加载完成后
2. 关键交互前后
3. 错误状态出现时
4. E2E 流程关键节点

### 命名规则
`<模块序号>-<页面>-<状态>.png`

### 存储位置
`<reportPath>/screenshots/`，报告中用相对路径 `screenshots/xxx.png` 引用

## 使用方式

```
用户: 测试 http://localhost:3000
→ 选择测试模块
→ 触发 playwright-e2e-tester Agent
→ 生成测试报告（含截图）: `test-reports/edu-web/report-<timestamp>.md`

用户: 测试 http://localhost:3000，报告放到 ./reports/test1/
→ 报告保存到指定路径
```

## 报告路径规则
- **默认**: `test-reports/<module>/report-<timestamp>.md`
- **自定义**: 用户指定任意路径（相对或绝对）
- **模板变量**: `<module>` 从URL提取（如 `edu-web`），`<timestamp>` 格式 `YYYYMMDDHHmmss`
- **截图目录**: 自动在报告目录下创建 `screenshots/` 子目录
