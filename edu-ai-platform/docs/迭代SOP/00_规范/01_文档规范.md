# 文档规范

> 本规范定义迭代SOP文档体系的命名规则、目录结构、格式标准。

## 1. 文档命名规范

### 1.1 命名格式

```
[序号]_[文档名称]_[版本].md

示例：
01_文档规范_1.0.md
02_迭代计划_模板_1.0.md
```

### 1.2 序号规则

| 序号 | 分类 |
|------|------|
| 00 | 索引与规范 |
| 01 | 模板库 |
| 02 | 角色定义 |
| 03 | AI边界 |
| 04 | 迭代流程 |
| 05 | 异常处理 |
| 06 | 附录 |

### 1.3 版本规则

- 初始版本：1.0
- 重大修订：X.0（X递增）
- 小幅修订：X.Y（Y递增）
- 仅供查阅的终版：X.Y-final

## 3. 文档格式规范

### 3.1 标题层级

```markdown
# 一级标题：文档标题
## 二级标题：主要章节
### 三级标题：子章节
#### 四级标题：细分内容
```

### 3.2 表格格式

```markdown
| 表头1 | 表头2 | 表头3 |
|-------|-------|-------|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

### 3.3 代码块

- 单行代码：`` `code` ``
- 多行代码：```` ```language ````

### 3.4 Mermaid图

- 流程图：```` ```mermaid flow TD ````
- 时序图：```` ```mermaid sequence ````
- 状态图：```` ```mermaid state ````

## 4. 文档元数据

### 4.1 必需元数据

每个文档开头应包含：

```markdown
> 文档标识：[唯一ID]
> 版本：1.0
> 更新日期：2026-01-01
> 维护人：[角色名称]
> 状态：[草稿/评审/已发布]
```

### 4.2 变更记录

文档末尾应包含变更记录：

```markdown
## 变更记录

| 版本 | 日期 | 变更人 | 变更说明 |
|------|------|--------|----------|
| 1.0 | 2026-01-01 | - | 初始版本 |
```

## 5. 文档编码

| 类型 | 编码 | 说明 |
|------|------|------|
| 中文 | UTF-8 | 无BOM |
| 标点 | 全角 | 中文文档使用全角标点 |
| 英文 | 半角 | 英文术语使用半角 |
