# 人工使用说明书

> 本说明书指导人类如何驱动AI数字员工完成迭代任务。

---

## 1. 快速开始

### 下达任务

使用 [AI任务指令模板](../01_模板库/AI任务指令_模板.md) 填写4项信息：

| 项目 | 说明 |
|------|------|
| 任务名称 | 如：用户登录API |
| AI角色 | 从列表选择 |
| 任务目标 | 1句话描述 |
| 关键产出 | 3-5项产出项 |

### 任务流转

```
下达任务 → AI自动读取上下文 → 执行任务 → 人类验收
```

### 任务状态

| 状态 | 说明 |
|------|------|
| 等待中 | 任务已下达，等待执行 |
| 执行中 | AI正在处理 |
| 待验收 | 产出等待人类验收 |
| 已完成 | 人类验收通过 |
| 已终止 | 任务被终止 |

---

## 2. 验收方法

### 各角色产出验收标准

| AI角色 | 产出类型 | 验收方式 | 验收标准 |
|--------|----------|----------|----------|
| **AI-PM** | 需求文档 | 业务确认 | 业务逻辑正确、完整 |
| **AI-UI** | 设计稿 | 视觉确认 | 符合设计规范 |
| **AI-Architect** | 架构方案 | 技术评审 | 技术可行、成本可控 |
| **AI-FE** | 前端代码 | 代码审查 | 编译通过、符合规范 |
| **AI-BE** | 后端代码 | 代码审查 | 编译通过、符合规范 |
| **AI-Test** | 测试用例 | 评审确认 | 覆盖度达标 |
| **AI-Writer** | 文档 | 内容审核 | 准确完整 |
| **AI-Reviewer** | 审查报告 | 人工复核 | 建议合理 |

### 反馈机制

| 结果 | 操作 |
|------|------|
| **通过** | 产出合格，进入下一环节 |
| **修改** | 产出需修改，明确修改意见后退回AI |
| **终止** | 产出严重偏离，终止任务转人类执行 |

### 审批时效

| 节点 | 审批人 | 时效要求 |
|------|--------|----------|
| 需求确认 | PM | 24小时内 |
| 技术方案 | 技术负责人 | 24小时内 |
| 代码合并 | 高级开发 | 实时 |
| 测试报告 | QA负责人 | 迭代内 |
| 文档归档 | 相关负责人 | 迭代内 |

## 3. 异常处理

### 干预触发条件

| 条件 | 触发动作 |
|------|----------|
| AI连续3次输出被拒绝 | 暂停AI执行 |
| AI执行超时 | 终止执行 |
| AI输出包含敏感信息 | 立即停止 |
| 人类发现明显错误 | 人工干预 |

### 干预操作

| 类型 | 操作 |
|------|------|
| **终止** | 立即停止AI |
| **修改** | 调整提示词，重新下达 |
| **补充** | 补充上下文信息 |
| **切换** | 切换人类执行 |

### 降级处理

| 级别 | 处理方式 |
|------|----------|
| **P0 阻断** | 立即停止 → 人工评估 → 修复或转人类 |
| **P1 警告** | 记录 + 尝试修正 → 成功则人工确认，失败则转人类 |
| **P2 提示** | 记录 + 展示 → 人类决策 |

## AI角色快速参考

| AI角色 | 核心能力 | 适用场景 |
|--------|----------|----------|
| **AI-PM** | 需求分析、优先级排序 | 需求梳理、用户故事编写 |
| **AI-UI** | 界面设计、视觉规范 | 界面设计 |
| **AI-Architect** | 技术架构设计 | 架构设计、技术选型 |
| **AI-FE** | 前端代码开发 | 页面开发、组件开发 |
| **AI-BE** | 后端API开发 | API开发、业务逻辑实现 |
| **AI-Test** | 测试用例编写、测试执行 | 用例生成、自动化测试 |
| **AI-Writer** | 技术文档、API文档 | 文档生成 |
| **AI-Reviewer** | 代码审查、安全扫描 | 代码审查 |

> 详细能力分级和边界判断规则见 [AI自主边界定义文档](../03_AI边界/01_AI自主边界定义.md)
