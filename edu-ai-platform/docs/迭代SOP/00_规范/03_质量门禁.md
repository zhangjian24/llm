# 质量门禁

> 本文档定义迭代过程中的质量检查标准和门禁规则。

## 1. 质量门禁体系概览

```mermaid
flowchart LR
    subgraph 准备阶段
        A1[需求评审] --> A2[技术方案]
    end
    
    subgraph 执行阶段
        A2 --> B1[代码开发]
        B1 --> B2[代码审查]
        B2 --> B3[单元测试]
    end
    
    subgraph 收尾阶段
        B3 --> C1[集成测试]
        C1 --> C2[验收测试]
        C2 --> C3[发布检查]
    end
    
    A1 -.- A2 -.- B1 -.- B2 -.- B3 -.- C1 -.- C2 -.- C3
    
    style A1 fill:#4caf50,color:#fff
    style A2 fill:#4caf50,color:#fff

```

## 2. 各阶段质量门禁

### 2.1 迭代准备阶段

| 检查点 | 检查项 | 标准 | 责任人 | 状态 |
|--------|--------|------|--------|------|
| 需求评审 | 需求完整性 | 100%覆盖用户故事 | PM | ⬜ |
| 需求评审 | 验收标准 | 每个需求有明确验收标准 | PM | ⬜ |
| 技术方案 | 技术可行性 | 通过技术评审 | Architect | ⬜ |
| 技术方案 | 风险识别 | 识别所有已知风险 | DEV | ⬜ |
| 任务估算 | 估算准确性 | 偏差≤30% | DEV | ⬜ |

## 5. 门禁执行流程

### 5.1 门禁检查流程

```mermaid
flowchart TD
    A[进入门禁检查点] --> B{检查项全部通过?}
    
    B -->|是| C[通过门禁]
    B -->|否| D{可修复?}
    
    D -->|是| E[修复]
    E --> A
    D -->|否| F[升级处理]
    
    C --> G[进入下一阶段]
    F --> H{影响迭代?}
    H -->|是| I[调整迭代范围]
    H -->|否| G
    
    style C fill:#4caf50,color:#fff
    style F fill:#f44336,color:#fff
    style I fill:#ff9800,color:#fff
```

### 5.2 门禁异常处理

| 异常类型 | 处理方式 | 审批人 |
|----------|----------|--------|
| 门禁检查失败 | 修复后重新检查 | 责任人 |
| 门禁豁免申请 | 评估后决定 | 技术负责人 |
| 门禁失败升级 | 调整迭代计划 | PM |

## 6. 质量报告

### 6.1 迭代质量报告

每个迭代结束需生成质量报告，包含：

| 内容 | 说明 |
|------|------|
| 质量指标达成 | 各指标实际值vs目标值 |
| 缺陷统计 | 数量、级别、趋势 |
| 代码质量 | 覆盖率、复杂度等 |
| AI执行质量 | AI产出合格率 |
| 改进建议 | 下迭代改进计划 |

### 6.2 质量趋势分析

| 周期 | 分析内容 |
|------|----------|
| 每周 | 代码质量趋势 |
| 每迭代 | 测试覆盖趋势 |
| 每月 | 缺陷密度趋势 |
| 每季度 | 整体质量评估 |
