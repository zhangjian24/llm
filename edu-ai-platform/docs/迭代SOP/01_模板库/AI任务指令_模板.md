# AI任务指令模板

> 本模板用于向AI数字员工下达任务。
> 人类只需填写4项核心信息，AI自动读取上下文并生成验收标准。
> AI必须从指定文档读取信息，不得要求人类重复提供已在文档中定义的内容。

---

| 项目 | 内容 | 说明 |
|------|------|------|
| **任务名称** | [任务名称，如：用户登录API] | 简洁的任务名称 |
| **AI角色** | [AI-PM/AI-FE/AI-BE/AI-Test/AI-Architect/AI-Reviewer/AI-Writer] | 执行任务的AI角色 |

| **关键产出** | [列出3-5项产出项] | 交付物清单 |

## AI自动读取区

### 读取流程（5步法）

```
步骤1 → 读取状态追踪文档 → 获取迭代上下文
步骤2 → 读取技术方案文档 → 获取技术栈、API设计、数据库设计
步骤3 → 读取需求文档 → 获取业务约束
步骤4 → 根据任务类型 → 自动生成验收标准
步骤5 → 合并通用约束 + 特有约束 → 完整约束列表
```

### 读取优先级

| 优先级 | 文档 | 必须读取 | 获取内容 |
|--------|------|----------|----------|
| P0 | 状态追踪文档 | ✅ | 迭代编号、目标、阶段、进度 |
| P1 | 技术方案文档 | ✅ | 技术栈、API设计、数据库设计、约束 |
| P2 | 需求文档 | ✅ | 业务约束、功能清单 |
| P3 | 历史任务产出 | 参考 | 上下文连续性 |

### 文档不存在时的处理

| 情况 | AI处理方式 |
|------|------------|
| 状态追踪文档不存在 | 提示创建，提供模板 |
| 技术方案文档不存在 | 提示先完成技术方案设计 |
| 需求文档不存在 | 提示先完成需求整理 |

## 验收标准（自动生成）

### 根据任务类型自动生成规则

| 任务类型 | 自动生成的验收标准 |
|----------|-------------------|
| **代码生成** | 1. 编译通过（mvn compile）<br>2. 单元测试通过（覆盖率≥70%）<br>3. 符合代码规范（静态检查）<br>4. API返回正确格式 |
| **代码审查** | 1. 代码扫描完成<br>2. 无高危安全问题<br>3. 无阻断Bug<br>4. 提供优化建议 |
| **测试执行** | 1. 测试用例执行率100%<br>2. 阻断缺陷0个<br>3. 严重缺陷0个<br>4. 生成测试报告 |
| **需求分析** | 1. 用户故事完整<br>2. 验收标准明确<br>3. 覆盖所有业务场景 |
| **技术方案** | 1. 架构设计合理<br>2. 技术风险识别完整<br>3. 方案可执行 |
| **文档生成** | 1. 内容完整准确<br>2. 格式规范<br>3. 图表清晰 |
